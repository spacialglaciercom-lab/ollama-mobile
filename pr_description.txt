🧪 Add tests for ollamaClient.ts checkServerStatus

🎯 What:
The `checkServerStatus` function in `src/api/ollamaClient.ts` lacked unit testing coverage.

📊 Coverage:
Added a new test suite `src/api/__tests__/ollamaClient.test.ts` which tests the following scenarios for `checkServerStatus`:
- Successful retrieval of models (200 OK)
- Correct handling of the Authorization header when an apiKey is provided
- Graceful handling when the response models array is missing/undefined
- Graceful status degradation (`status: false`) when the response is not `ok`
- Graceful failure on network errors (fetch promise rejection)

✨ Result:
Improved test coverage and reliability for Ollama server API requests by explicitly mocking and asserting external network interactions via the global fetch API.
