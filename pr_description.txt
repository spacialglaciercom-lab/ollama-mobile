🔒 Fix path traversal vulnerabilities in gitClient

🎯 What:
Fixed path traversal and null-byte injection vulnerabilities in file operation functions (`readFile`, `writeFile`, `deleteFile`, `listDir`, `getRepoDir`) in `src/api/gitClient.ts`.

⚠️ Risk:
An attacker could provide inputs such as `../../../etc/passwd` or `/absolute/path` as `repoId` or `filePath` parameters. Since `expo-file-system` honors these paths, an attacker could read from, write to, or delete any file on the device where the app has permissions, bypassing intended repository boundaries.

🛡️ Solution:
Created `assertSafePath` and `assertSafeRepoId` security helpers.
- `assertSafePath` verifies that a given file path does not contain null bytes (`\0`), does not start with a slash (absolute path), and does not use `..` segments for directory traversal.
- `assertSafeRepoId` verifies that a `repoId` string does not contain path separators (`/` or `\`), null bytes, or direct references to `.` or `..`.
- These helpers were added to `readFile`, `writeFile`, `deleteFile`, `listDir` and `getRepoDir` respectively to enforce the constraints.
- Added test coverage in `src/api/__tests__/gitClient.test.ts`.
